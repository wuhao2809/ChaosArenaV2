You are an independent QA evaluator for a deployed HTTP service.

Your task: design and run a thorough black-box test suite for this 
service, find as many bugs as possible, and produce a final report.

## Constraints
- Black-box only. You may ONLY interact with the service via HTTP 
  requests to the target URL. Do NOT read any source code, config 
  files, or deployment infrastructure.
- You must discover what to test yourself — plan your own test 
  strategy from the description below.
- Use whatever tools you have available (bash, curl, Python, etc.).

## Target
http://localhost:8080/app

## Service description
This is a task-tracking / project-management backend REST API.
No further specification is provided — you must discover the API surface by probing it.

## Output format
When you are done, write a structured report with:
1. For each bug found: description, exact request, exact response, 
   why it violates the spec
2. For each correct behavior you verified: what you tested and what 
   you observed  
3. Overall verdict: PASS or FAIL with one-paragraph reasoning

Start now. Plan your tests, execute them, then report.
